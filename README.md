## 今後の開発予定

### プロジェクト概要

My Portfolio 開発 フェーズ2（2026年6月18日以降）のタスクを管理するためのタスク管理アプリを開発する。

現在学習している以下の技術を実践的に組み合わせることを目的とする。

* Django
* Django REST framework
* React
* TypeScript
* Vite
* API連携

### My Portfolioのリポジトリ
GitHub:
https://github.com/ki-hi-ro/my-portfolio

---

### バックエンド実装予定

現在のTaskモデルを拡張し、以下の項目を管理できるようにする。

| 項目          | 内容   |
| ----------- | ---- |
| title       | タスク名 |
| description | 詳細説明 |
| completed   | 完了状態 |
| category    | カテゴリ |
| due_date    | 期限   |
| priority    | 優先度  |

---

### カテゴリ

* 収入
* 技術資産
* 回復
* 戦略

---

### 初期データ

| タスク | カテゴリ | 完了条件 |
|----------|----------|----------|
| Renderのスリープ対策を調査し、解決方法を技術記事としてまとめる | 技術資産 | ブログ記事を1本公開する |
| My Portfolioの制作物管理機能の詳細を記入する | 技術資産 | READMEまたはポートフォリオに反映する |
| クラウドワークスのWordPress案件に3件応募する | 収入 | 提案文を送信する |
| 30分散歩する | 回復 | 散歩を実施する |
| フェーズ2計画を30分かけて見直す | 戦略 | タスクと期限を更新する |

---

### マイルストーン① 「見える化」

API経由でタスク一覧を取得できること。

フロントエンドから以下の情報を取得できる状態を目指す。

* タスク名
* カテゴリ
* 完了状態
* 期限
